Quick Start

1. Installation

# Download TMC curl -O https://releases.tmc.ai/tmc-v1.0.0.tar.gz tar -xzf tmc-v1.0.0.tar.gz cd tmc-v1.0.0 # Build (Rust required) ./build.sh # Or download pre-built binary curl -O https://releases.tmc.ai/tmc-server-linux-x64 chmod +x tmc-server-linux-x64

2. Get Your License Key

Purchase a license at tmc.ai and you'll receive:

TMC-A7F3-9B2C-4D8E-1F6A

3. Start the Server

TMC_LICENSE_KEY="TMC-XXXX-XXXX-XXXX-XXXX" ./tmc-server # Output: # üöÄ TMC Server v1.0.0 # üîê Validating license... # ‚úÖ License valid (Your Company, expires 2026-12-25) # üîì Activating license... # ‚úÖ Activation successful (1/3 used) # ‚úÖ TMC Server ready on http://127.0.0.1:8000

API Reference

Crystallize Memory

Store knowledge in geometric crystal structure.

POST /crystallize Content-Type: application/json { "text": "Kent Stone created TMC in 2025", "importance": 0.9 } Response: { "node_id": "12abc...", "success": true }

Retrieve Memories

Query the crystal lattice for relevant memories.

POST /retrieve Content-Type: application/json { "query": "Who created TMC?", "k": 5 } Response: { "results": [ { "node_id": "12abc...", "content": "Kent Stone created TMC in 2025", "score": 0.94, "importance": 0.9 } ], "count": 1 }

Get Statistics

GET /stats Response: { "stats": { "total_nodes": 15234 } }

Integration Examples

Python

import requests # Store memory response = requests.post("http://localhost:8000/crystallize", json={ "text": "User prefers dark mode", "importance": 0.7 }) # Retrieve memory response = requests.post("http://localhost:8000/retrieve", json={ "query": "user interface preferences", "k": 5 }) memories = response.json()["results"]

JavaScript

// Store memory const response = await fetch('http://localhost:8000/crystallize', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ text: 'User speaks English and Spanish', importance: 0.8 }) }); // Retrieve memory const results = await fetch('http://localhost:8000/retrieve', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ query: 'language preferences', k: 3 }) }).then(r => r.json());

cURL

# Store curl -X POST http://localhost:8000/crystallize \ -H "Content-Type: application/json" \ -d '{"text": "Meeting at 3pm tomorrow", "importance": 0.95}' # Retrieve curl -X POST http://localhost:8000/retrieve \ -H "Content-Type: application/json" \ -d '{"query": "upcoming meetings", "k": 5}'

Advanced Configuration

Environment Variables

Variable Description Default
TMC_LICENSE_KEY Your license key (required) -
TMC_PORT Server port 8000
TMC_HOST Bind address 127.0.0.1
TMC_DIMENSION Embedding dimension 384

Production Deployment

# Run as systemd service sudo cat > /etc/systemd/system/tmc.service << 'EOF' [Unit] Description=TMC Server After=network.target [Service] Type=simple User=tmc Environment="TMC_LICENSE_KEY=TMC-XXXX-XXXX-XXXX-XXXX" Environment="TMC_HOST=0.0.0.0" Environment="TMC_PORT=8000" ExecStart=/opt/tmc/tmc-server Restart=always [Install] WantedBy=multi-user.target EOF sudo systemctl enable tmc sudo systemctl start tmc

Performance Tips

üí° Tip: TMC is already optimized for maximum performance (100-200¬µs latency), but you can:
  • Use higher importance values (0.7-1.0) for critical memories
  • Keep queries specific for best retrieval accuracy
  • Batch crystallize operations when storing multiple memories

License Management

Activation Limits

Your license tier determines how many servers can run simultaneously:

  • Starter: 1 activation
  • Pro: 3 activations
  • Enterprise: Unlimited

Deactivating a Server

# Free up an activation slot TMC_LICENSE_KEY="TMC-XXXX-XXXX-XXXX-XXXX" ./tmc-server --deactivate
‚ö†Ô∏è Important: Each server activation is machine-locked via hardware fingerprint. Moving to a new server requires deactivating the old one first (unless you have spare activation slots).
