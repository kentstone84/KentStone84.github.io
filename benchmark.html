Methodology

Test Configuration

  • Date: December 25, 2025
  • Hardware: M-series MacBook Pro (production hardware)
  • Dataset: 100,000 synthetic memories
  • Memory Size: Average 50-100 characters per memory
  • Retrieval Iterations: 200 queries per system
  • Query Type: Semantic similarity search

TMC Configuration

  • Version: TMC Core v1.0.0
  • Dimension: 384 (default)
  • Batch Size: Unlimited (loaded all 100K at once)
  • Chunking: None required

ChromaDB Configuration

  • Version: ChromaDB 0.4.x
  • Embedding: Default all-MiniLM-L6-v2
  • Batch Size: 5,000 (required chunking into 20 batches)
  • Distance: Cosine similarity

Metrics Collected

  • Ingestion Time: Total time to load all 100K memories
  • Throughput: Operations per second during ingestion
  • Retrieval Latency: Mean, median, P95, P99 over 200 queries
  • Memory Usage: Peak RAM during operations

Reproducibility

We believe in transparent, reproducible benchmarks. Here's everything you need to run this test yourself. 

# Clone benchmark repository
git clone https://github.com/kentstonedev/tmc-benchmarks
cd tmc-benchmarks

# Install dependencies
pip install -r requirements.txt

# Run benchmark
python benchmark_tmc.py

# Expected output:
# TMC: 43s load, 0.60ms retrieval
# ChromaDB: 1559s load, 67.57ms retrieval
# 113x faster retrieval

System Requirements

  • RAM: Minimum 8GB (16GB recommended)
  • Disk: 10GB free space
  • Python: 3.8 or higher
  • Time: ~30 minutes to complete full benchmark
